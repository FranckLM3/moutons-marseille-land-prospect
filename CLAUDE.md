## Project Overview

Geospatial pipeline to identify grazeable land parcels in the Aix-Marseille-Provence (AMP) metropolitan area. Produces a Leaflet interactive map deployed on GitHub Pages. The unit of output is a **cadastral parcel** enriched with a `pct_prairie` field (% grassland coverage from OCS GE land cover data).

## Key Commands

### Setup
```bash
uv venv && uv pip install -r requirements.txt
# or
python -m venv .venv && source .venv/bin/activate && pip install -r requirements.txt
```

Always use `.venv/bin/python` to run scripts (hardcoded in `build_and_import_paca.py`).

### Build pipeline (dept 13 — AMP)
```bash
python scripts/build.py                                     # full pipeline with defaults
python scripts/build.py --min-area 5000                     # filter parcels ≥ 5000 m²
python scripts/build.py --split-communes                    # also write per-commune GeoJSON + index.json
python scripts/build.py --inject-only                       # regenerate docs/index.html from template only
```

The `--split-communes` flag writes one GeoJSON per commune to `docs/data/communes/` plus an `index.json` with bbox metadata. These files are committed to git and served statically — no Supabase required for data loading.

### Build for all PACA departments
```bash
python scripts/download_paca.py                # download cadastre for non-13 depts
python scripts/build_and_import_paca.py        # build + Supabase import, all depts
python scripts/build_and_import_paca.py --dept 83 84 --build-only
```

### Supabase import (legacy — data now served statically)
```bash
python scripts/import_parcelles_supabase.py --geojson docs/pasture_zones.geojson
python scripts/import_parcelles_supabase.py --dry-run
```

### Serve locally
```bash
python -m http.server 8000 --directory docs --bind 127.0.0.1
# → http://127.0.0.1:8000
```

## Architecture

### Pipeline flow (scripts/build.py)
```
[1/5] load_ocsge()              → GeoPackage → EPSG:4326
[2/5] filter_pasture_zones()    → CS2.2.1 (herbacées) + CS2.1.2 (landes/maquis) + others
[3/5] load_cadastre()           → Etalab parcels (Marseille = 16 arrondissements 13201-13216)
      join_owners_to_cadastre() → left join on parcel ID (keeps parcels without owner by default)
[4/5] add_prairie_ratio()       → spatial intersection OCS GE × cadastre → pct_prairie
[5/5] export_geojson()          → docs/pasture_zones.geojson (simplified to 2m)
      export_by_commune()       → docs/data/communes/{slug}.geojson + index.json (if --split-communes)
      _inject_ors_key()         → docs/index.html from template + .env keys
```

### Source modules (src/)
- `land_cover.py` — loads OCS GE GeoPackage, filters by coverage codes
- `filters.py` — area filtering (`area_m2`, `area_ha` columns)
- `cadastre.py` — loads cadastre (Etalab), `add_prairie_ratio()` via spatial intersection; contains `AMP_COMMUNE_NAMES` dict for all AMP communes
- `owners.py` — loads `parcelles-des-personnes-morales.geojson`, joins to cadastre
- `export.py` — geometry simplification + GeoJSON export; `export_by_commune()` splits by commune with `_slugify()` for filenames

### Frontend (docs/)
- `index.template.html` — HTML template with `__ORS_KEY__`, `__SUPABASE_URL__`, `__SUPABASE_ANON_KEY__` placeholders
- `app.js` — Leaflet map, route calculation via OpenRouteService, contributor feedback (localStorage + Supabase), commune/area/owner type filters; data loaded from static GeoJSON files (no Supabase for parcel data)
- `app.css` — UI styles
- `docs/index.html` — **git-ignored**, generated by `_inject_ors_key()` from the template
- `docs/data/communes/` — static GeoJSON files per commune + `index.json`; tracked in git via `.gitignore` exception

### Data loading (frontend)
Parcels are served as static files — no Supabase required for map data:
- `data/communes/index.json` — `[{name, file, bbox}]` per commune, loaded at startup
- `data/communes/{slug}.geojson` — GeoJSON FeatureCollection for each commune, lazy-loaded on demand
- `communeCache` in `app.js` caches loaded communes across filter changes
- Corridor search uses `_findCorridorInMemory()`: auto-loads communes whose bbox intersects the route, then filters by `distToPolyline()` — no PostGIS RPC needed
- Supabase is kept only for contributor feedback (`parcel_feedback`, `parcel_comments`)

### Data sources
- **OCS GE IGN** — GeoPackage, must be downloaded manually from geoservices.ign.fr (auth required); placed at `data/ocsge/<department path>/OCCUPATION_SOL.gpkg`
- **Cadastre Etalab** — downloaded automatically by `scripts/download_paca.py`, cached in `data/cadastre/`
- **Parcelles des personnes morales** — `data/parcelles-des-personnes-morales.geojson` (from Koumoul)

### Environment variables (.env — never committed)
```
ORS_API_KEY=           # OpenRouteService (routing), 2000 req/day free
SUPABASE_URL=          # optional, for contributor feedback sync only
SUPABASE_ANON_KEY=     # used by frontend (read/write feedback)
SUPABASE_SERVICE_KEY=  # used by import_parcelles_supabase.py (admin writes)
```

### Deployment
Two git remotes: `origin` (Gitea) and `github` (GitHub). GitHub Actions workflow (`.github/workflows/deploy.yml`) builds `docs/index.html` with secrets and deploys `docs/` to GitHub Pages. Push to both remotes manually. Commune GeoJSON files in `docs/data/communes/` are committed to git and deployed automatically.

## Data Notes

- Marseille cadastre is split into 16 arrondissements (INSEE codes 13201–13216), handled specially in `load_cadastre()`
- OCS GE coverage codes for prairie: `CS2.2.1` (formations herbacées) and `CS2.1.2` (landes/maquis); additional woody codes exist but are filtered differently
- `pct_prairie` = (OCS GE grassland area ∩ parcel area) / parcel area × 100
- Commune slugs use `_slugify()` (NFKD normalization, lowercase, underscores); `index.json` maps slug → commune name + bbox
- Corridor search is now purely in-memory: bbox intersection to select communes to load, then `distToPolyline()` per feature centroid
